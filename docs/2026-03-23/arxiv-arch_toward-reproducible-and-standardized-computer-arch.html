Toward Reproducible and Standardized Computer Architecture Simulation with gem5

arXiv Computer Architecture 2026-03-23
Read Original Article →

论文摘要 / Abstract

本文针对基于模拟的计算机架构研究中可重复性不足的问题，对开源模拟器gem5及其资源库进行了系统性改进。研究识别了三大核心挑战：磁盘镜像创建缺乏标准化、客户机-宿主机通信能力受限、多工作负载模拟需要复杂的外部脚本。为此，作者引入了基于Packer的跨ISA（x86/ARM/RISC-V）磁盘镜像标准化流程、重构的基于类的退出事件系统与超调用机制、以及支持并行全系统模拟的Suites和MultiSim功能，显著提升了模拟实验的可重复性和标准化程度。

方法概述 / Method

论文采用基础设施增强的方法论，首先通过Packer工具实现跨三种主流ISA（x86、ARM、RISC-V）的磁盘镜像创建流程标准化，并提供预注释的基准测试套件；其次将gem5原有的预定义退出事件系统重构为可扩展的类架构，引入超调用（hypercalls）机制以实现动态的客户机-宿主机通信；最后开发Suites和MultiSim模块，使多工作负载并行模拟能够直接在gem5配置脚本中完成，无需依赖外部协调脚本。

实验结果 / Results

研究成果包括交付12个新磁盘镜像、6个新内核以及覆盖三种ISA的200余个工作负载，并配套提供gem5-bridge驱动程序和远程监控工具。这些资源经过验证且可直接复用，从根本上解决了此前"创建自定义磁盘镜像复杂耗时"和"多模拟协调脚本易出错"的实践痛点，使研究人员能够基于标准化、可扩展的资源快速开展实验。

应用价值 / Applications

该研究对计算机架构学术界和工业界具有重要价值：为超过1600个gem5分支的开发者社区提供了可重复、可共享的实验基础设施，降低新研究者的入门门槛；标准化资源库支持跨研究团队的基准测试结果对比，促进科学发现的验证与累积；企业研发部门可利用这些工具进行处理器设计的前期评估，减少因环境配置差异导致的决策偏差。整体而言，该工作推动了计算机架构模拟研究向更开放、透明和高效的方向发展。
