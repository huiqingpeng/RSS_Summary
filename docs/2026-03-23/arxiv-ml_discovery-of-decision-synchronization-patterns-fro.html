Discovery of Decision Synchronization Patterns from Event Logs

arXiv Machine Learning 2026-03-23
Read Original Article →

论文摘要 / Abstract

本文提出了一种从事件日志中发现决策同步模式的方法，以解决业务流程中多个案例之间决策协调的问题。现有流程发现技术主要关注单个案例的属性，而本文方法能够识别案例间的同步机制，这些机制通过延迟某些案例来优先执行其他案例，从而实现资源的公平高效利用。作者形式化了四种决策同步模式，并展示了如何从事件日志中自动发现这些模式的约束条件。通过在人工场景中的评估，验证了该方法能够可靠地识别各类决策同步模式。

方法概述 / Method

该方法借鉴供应链流程的特点，将决策同步模式定义为特定流程构造与案例选择约束的组合。针对四种不同的决策同步模式，作者分别设计了相应的发现算法，通过分析事件日志中案例的执行顺序、资源分配和时间关系，推断出控制案例优先级的约束条件。这些约束决定了在多个竞争案例中选择执行哪一个案例。

实验结果 / Results

实验在两个场景中进行验证：第一个场景使用四个独立的流程模型，每个模型包含单一类型的决策同步模式，结果表明该方法能够准确发现每种类型的模式；第二个场景采用包含全部四种决策同步模式的复杂流程模型，证明了方法的泛化能力。两个场景均成功可靠地恢复了预期的决策同步模式。

应用价值 / Applications

该研究可应用于业务流程挖掘和优化，帮助组织识别现有流程中隐含的决策同步机制，从而改进资源调度和案例优先级管理。特别适用于需要协调多个并行案例的场景，如客户服务、医疗预约、供应链管理等，有助于提升流程的公平性和效率，减少不必要的等待时间。
