KV Cache Quantization for Self-Forcing Video Generation: A 33-Method Empirical Study

arXiv Machine Learning 2026-03-31
Read Original Article →

论文摘要 / Abstract

本论文针对自强制视频生成（self-forcing video generation）中的KV缓存内存瓶颈问题，开展了涵盖33种量化与缓存策略变体的大规模实证研究。研究基于Wan2.1自强制架构，在MovieGen和StoryEval两个评测场景下，综合评估了峰值显存、运行时间、压缩比、图像质量及终端漂移等指标。核心发现表明：FlowCache启发的软剪枝INT4自适应方法实现了5.42-5.49倍压缩率，将峰值显存从19.28 GB降至约11.7 GB，且仅带来适度运行时开销；而高保真压缩方法（如PRQ_INT4和QUAROT_KV_INT4）因严重的运行时间或内存代价并非最佳部署选择；此外，名义压缩比本身并不充分，部分方法因注意力计算和刷新阶段仍需重建或保留大尺寸BF16缓冲区而未能降低实际峰值显存。

方法概述 / Method

研究构建了一个基于Wan2.1的自强制视频生成技术栈，系统性地对比了33种KV缓存量化与缓存策略变体，涵盖INT4/INT8量化、旋转量化、剪枝策略及其组合。实验设计包含610个提示级别的观测和63个基准级别的汇总，采用VBench成像质量指标、BF16参考保真度指标（SSIM、LPIPS、PSNR）以及终端漂移等多维度评估体系，同时精确测量峰值显存占用与实际压缩比。

实验结果 / Results

FlowCache启发的软剪枝INT4自适应方法在单镜头10秒生成（MovieGen）和长叙事稳定性（StoryEval）场景下均表现最优，实现5.42-5.49倍压缩，峰值显存降低至约11.7 GB（降幅约39%），运行时间开销可控。相比之下，PRQ_INT4和QUAROT_KV_INT4等方法虽保真度更高，但运行时间显著增加或内存代价过高。研究还发现，部分方法虽名义压缩比高，却因架构集成问题（BF16缓冲区重建/保留）未能降低实际峰值显存，揭示了当前KV缓存压缩与注意力机制集成中的关键瓶颈。

应用价值 / Applications

本研究为视频生成模型的实际部署提供了系统性的KV缓存压缩选型指南，尤其适用于资源受限环境下生成长视频内容的场景。研究成果可直接指导Wan2.1等开源视频生成模型的工程优化，帮助开发者在质量-速度-内存三维度间做出权衡决策。此外，论文公开的数据产品、分析工作流和评测框架为后续KV缓存量化研究提供了标准化基准，推动更高效的内存集成架构设计。
