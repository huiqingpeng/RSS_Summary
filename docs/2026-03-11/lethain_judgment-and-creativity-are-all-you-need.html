是什么 / What it is

这篇文章探讨了AI编程助手（coding agents）如何从根本上改变软件开发的工作模式。作者通过对比2014年Uber的技术迁移与2024/2025年Imprint的迁移经历，展示了从"手工实现决策"到"专注设计决策"的转变——前者需要工程师编写大量代码，后者则主要由AI生成代码，人类聚焦于判断和创意。
This article explores how AI coding agents are fundamentally transforming software development workflows. By contrasting Uber's 2014 technology migration with Imprint's 2024/2025 migration, the author illustrates the shift from "manually implementing decisions" to "focusing on designing decisions"—where engineers previously wrote extensive code, but now AI generates code while humans concentrate on judgment and creativity. ---

为什么重要 / Why it matters

这标志着软件开发瓶颈的根本转移：从"时间和注意力受限"转向"判断力和创造力受限"。当AI解决了执行层面的效率问题，工程师的核心价值将体现在架构设计、风险评估和长期可维护性决策上——这些正是难以自动化的"判断"能力。这一趋势将重塑技术团队的人才结构和技能要求。
This marks a fundamental shift in software development bottlenecks: from "time and attention constrained" to "judgment and creativity constrained." As AI solves execution-level efficiency, engineers' core value will manifest in architectural design, risk assessment, and long-term maintainability decisions—these "judgment" capabilities are precisely what resist automation. This trend will reshape talent structures and skill requirements for technical teams. ---

我能用什么 / How I can use it

立即开始实践"判断优先"的工作方式：将复杂任务拆解为设计文档和决策框架，再交由AI执行；同时建立可复用的"技能包"（skills/packages）来沉淀团队的最佳实践——如安全规范、代码标准等，作为AI的上下文输入。最终目标是培养自己从"写代码的人"进化为"定义问题、评估方案、把控质量的人"。
Start practicing "judgment-first" workflows immediately: decompose complex tasks into design documents and decision frameworks, then delegate execution to AI; simultaneously build reusable skill packages to capture team best practices—such as security standards and code guidelines—as context for AI. The ultimate goal is evolving yourself from "someone who writes code" to "someone who defines problems, evaluates solutions, and controls quality."
