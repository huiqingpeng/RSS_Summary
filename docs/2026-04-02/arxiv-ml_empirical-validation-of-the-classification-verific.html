Empirical Validation of the Classification-Verification Dichotomy for AI Safety Gates

arXiv Machine Learning 2026-04-02
Read Original Article →

论文摘要 / Abstract

本研究通过大规模实证实验，证明了基于分类器的AI安全门（safety gates）在AI系统自我改进过程中存在结构性失效问题。研究发现，18种不同的分类器配置（包括MLP、SVM、随机森林、k-NN、贝叶斯分类器和深度网络）以及3种安全强化学习基线方法，均无法满足安全自我改进的双重条件。然而，这种不可能性仅限于分类方法本身——基于Lipschitz球的验证器（verifier）能够在多个维度规模（最高至d=17,408）上实现零误接受率，并通过球链（ball chaining）技术实现无界参数空间遍历，为AI安全自我改进提供了可行的替代路径。

方法概述 / Method

研究采用对比实验设计，系统评估了分类器与验证器两类安全门机制。对于分类器，测试了涵盖传统机器学习到深度学习的18种配置；对于验证器，设计了基于解析Lipschitz界的球形验证方法，并开发了"球链"技术实现参数空间的连续安全扩展。实验在自改进神经控制器（d=240）和MuJoCo基准环境（最高d=1824）以及大语言模型LoRA微调场景中进行验证。

实验结果 / Results

分类器在安全分布分离度δ_s=2.0受控条件下全部失效，包括训练准确率100%的MLP和NP最优检验。相比之下，Lipschitz球验证器在d∈{84, 240, 768, 2688, 5760, 9984, 17408}的所有维度上实现δ=0的零误接受。在MuJoCo Reacher-v4上，10个球链带来+4.31奖励提升；在Qwen2.5-7B-Instruct的LoRA微调中，42次链式转换跨越234倍单球半径，200步内零安全违规。组合式分组验证使半径扩展至全网络球的37倍。

应用价值 / Applications

该研究为AI系统的安全自我改进提供了关键的理论边界和实践指导：明确警示了当前工业界广泛采用的分类器安全门存在根本性局限，同时为高维参数空间中的安全优化（如大模型微调、强化学习策略迭代）提供了可扩展的验证框架。基于Lipschitz验证的方法可应用于需要严格安全保证的自主系统、金融交易算法、医疗AI等高风险场景，支持在持续学习或自我改进过程中维持形式化安全约束。
