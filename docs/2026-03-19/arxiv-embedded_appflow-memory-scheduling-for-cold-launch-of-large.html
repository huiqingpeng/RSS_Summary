AppFlow: Memory Scheduling for Cold Launch of Large Apps on Mobile and Vehicle Systems

arXiv Embedded Systems 2026-03-19
Read Original Article →

论文摘要 / Abstract

本文针对GB级大型应用（如端侧大语言模型和富媒体编辑器）在移动和车载系统中冷启动缓慢的问题，提出了系统级解决方案。研究发现86.6%的GB级应用冷启动超过1秒的用户体验临界点，而现有优化技术相互冲突（预加载消耗内存、内存回收破坏预加载收益）。基于"应用文件访问模式可预测"的核心洞察，作者设计了AppFlow系统，通过整合选择性文件预加载、自适应内存回收和上下文感知进程终止三大组件，在不修改应用的前提下将冷启动延迟降低66.5%，并实现95%的启动在1秒内完成。

方法概述 / Method

AppFlow采用预测驱动的系统级调度架构，跨Android框架层和Linux内核层实现。其核心包含三个协同组件：选择性文件预加载器（基于文件访问模式预测预加载关键数据）、自适应内存回收器（智能回收不破坏预加载收益的内存）、上下文感知进程终止器（选择性终止进程以平衡内存释放与后台存活率）。三者通过统一调度协调，解决现有优化技术孤立设计导致的冲突问题。

实验结果 / Results

实验表明AppFlow将GB级应用冷启动延迟从2秒降至690毫秒（降幅66.5%），在100天长期测试中保持95%的启动在1秒内完成。相比现有方案，该系统显著提升了响应速度和多任务体验，同时改善了后台应用存活率，避免了反复冷启动的恶性循环。

应用价值 / Applications

该研究直接适用于高端智能手机、车载信息娱乐系统等资源受限的移动计算平台，为端侧大模型、AR/VR应用、视频编辑器等重型应用的流畅运行提供系统级支撑。其价值在于弥合了Android Vitals 5秒慢启动阈值与用户体验1秒临界点之间的满意度鸿沟，对下一代GB级应用生态的普及具有关键推动作用。
