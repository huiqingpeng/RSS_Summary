From 8086 to Z80: Building a NASM-Inspired SDK for 8-Bit Retro Computing

Hackaday 2026-03-17
Read Original Article →

是什么 / What it is

HC SDK 是一个由 Humberto Costa 开发的开源汇编器工具链项目，旨在将 NASM（Netwide Assembler）风格的 Intel 语法扩展到 8 位复古处理器（如 Z80、8080、8085、8086），让开发者能在现代平台（macOS、Windows、Linux、BSD）上用熟悉的语法编写老式系统的汇编代码。
HC SDK is an open-source assembler toolchain developed by Humberto Costa that extends NASM-style Intel syntax to 8-bit retro processors (Z80, 8080, 8085, 8086), allowing developers to write assembly for vintage systems using familiar syntax on modern platforms (macOS, Windows, Linux,BSD). ---

为什么重要 / Why it matters

复古计算和自制计算机社区正在复兴，但不同架构的汇编器语法碎片化严重，学习成本高；HC SDK 通过统一 NASM 语法标准，降低了跨平台复古开发的门槛，并计划扩展高级语言支持，有望成为复古系统开发的完整解决方案。
The retro computing and homebrew computer community is experiencing a revival, yet assembler syntax fragmentation across architectures creates steep learning curves; HC SDK lowers barriers to cross-platform retro development by unifying NASM syntax standards, with planned high-level language support positioning it as a comprehensive solution for vintage system development. ---

我能用什么 / How I can use it

若你从事复古游戏开发、FPGA 软核实现或教育用途的 8 位计算机项目，可直接从 GitHub 获取 HC SDK 替代原生汇编器；同时可关注其 6502 支持进展及未来高级语言集成，评估是否迁移现有复古代码库以统一工具链。
If you're working on retro game development, FPGA soft-core implementations, or educational 8-bit computer projects, obtain HC SDK from GitHub to replace native assemblers; also monitor its 6502 support progress and future high-level language integration to evaluate migrating existing retro codebases for toolchain unification.
