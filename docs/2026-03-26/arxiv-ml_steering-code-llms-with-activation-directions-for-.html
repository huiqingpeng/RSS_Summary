Steering Code LLMs with Activation Directions for Language and Library Control

arXiv Machine Learning 2026-03-26
Read Original Article →

论文摘要 / Abstract

本文研究了代码大语言模型（Code LLMs）在生成代码时对特定编程语言和库的偏好是否以近似线性方向的形式编码在激活空间中。作者使用均值差分法估计了五组语言/库对的逐层引导向量，并在推理时将其添加到模型隐藏状态中。实验表明，这种干预能显著增强模型在中性提示下生成目标生态系统的代码，甚至在提示明确要求相反选择时仍保持一定效果，揭示了代码风格偏好具有可操控的紧凑结构。

方法概述 / Method

作者采用**均值差分法（difference-in-means）**来估计层级的引导向量（steering vectors）：分别计算模型在处理目标语言/库代码与非目标代码时各层激活状态的均值差异，得到方向向量。在推理阶段，通过将这些向量以不同强度系数添加到对应层的隐藏状态，实现对生成内容的定向引导。

实验结果 / Results

在三个开源代码LLM上的实验显示：引导干预能大幅提升中性提示下目标生态系统的生成比例；引导效果因模型和目标而异，主流生态系统（如Python）比小众替代方案更容易诱导；过强的干预系数会降低输出质量（如语法正确性）。值得注意的是，即使提示明确要求使用相反语言/库，引导向量仍能保持一定效果。

应用价值 / Applications

该研究为**可控代码生成**提供了轻量级、无需微调的技术路径，可用于：IDE中根据项目上下文自动适配语言/库偏好、抑制模型对特定生态系统的偏见、或在多语言代码库中灵活切换生成风格。该方法仅修改推理时的激活状态，计算开销低，便于集成到实际开发工具中。
