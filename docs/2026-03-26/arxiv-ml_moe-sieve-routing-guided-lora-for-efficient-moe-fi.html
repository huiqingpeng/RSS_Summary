MoE-Sieve: Routing-Guided LoRA for Efficient MoE Fine-Tuning

arXiv Machine Learning 2026-03-26
Read Original Article →

论文摘要 / Abstract

本文发现MoE模型中专家路由呈现高度偏斜分布：少数专家处理大部分token，而多数专家很少被激活（"冷专家"）。基于此，作者提出MoE-Sieve框架，仅对每层top-k高频路由专家应用LoRA微调。实验表明，仅微调25%的专家即可保持与全量LoRA相当的性能（差异在±1%以内），同时大幅减少可训练参数和训练时间。研究还揭示了适应冷专家会引入梯度噪声而不提升精度的现象。

方法概述 / Method

MoE-Sieve采用路由引导的稀疏LoRA策略：首先在小规模校准集上分析各层专家的路由频次，然后每层仅选择top-k高频专家（如25%）添加LoRA适配器，其余专家保持冻结。该方法通过路由统计信号实现专家选择，而非随机或贪婪预算优化。

实验结果 / Results

在两种不同架构的MoE模型和三个多样化任务上，MoE-Sieve将LoRA可训练参数减少70-73%，适配器检查点大小减少71-73%，墙钟训练时间最高减少50%，同时保持与全量LoRA相当的准确率（平均差异±1%）。随机专家选择在相同预算下性能下降约2.5个百分点，证明路由信号的重要性；而逐层贪婪预算优化相比均匀top-k无显著提升。

应用价值 / Applications

MoE-Sieve可显著降低大规模MoE模型的微调成本，适用于资源受限场景下的模型定制，如领域适配、指令微调等。该方法为MoE模型的高效部署提供实用方案，尤其在需要频繁微调或存储多个任务适配器的应用中具有重要价值。
