Per-Bank Memory Bandwidth Regulation for Predictable and Performant Real-Time System

arXiv Computer Architecture 2026-03-30
Read Original Article →

论文摘要 / Abstract

本文揭示了DRAM干扰本质上是银行级别的现象，而非传统的按核心带宽调节所假设的单片式资源。作者表征了现代DRAM的保证带宽，发现其在不同代际间保持基本恒定，并展示了单银行攻击如何利用这一限制。基于此，作者设计并实现了首个开源RISC-V SoC中的按银行内存带宽调节器，在保持实时工作负载相同性能隔离保证的同时，将尽力而为工作负载的平均吞吐量提升了5.74倍。

方法概述 / Method

作者采用FireSim仿真平台，在开源RISC-V SoC中设计实现了按银行（per-bank）内存带宽调节机制，通过细粒度地控制每个DRAM银行的访问而非将内存视为整体资源，实现对银行级并行性的感知与管理。该方法结合了对现代DRAM保证带宽特性的表征分析，以及针对银行级干扰的调节策略设计。

实验结果 / Results

评估表明，按银行调节有效缓解了对抗性银行争用，相比传统的银行无感知（bank-oblivious）方法实现了5.74倍的平均吞吐量提升；同时，该方法为实时工作负载提供了与传统方法同等水平的性能隔离保证。实验综合使用了合成工作负载和真实世界工作负载进行验证。

应用价值 / Applications

该研究对需要严格实时保证的嵌入式系统、汽车电子、工业控制等领域具有重要价值，能够在多核SoC共享DRAM的场景下同时保障关键任务的确定性延迟和提升非关键任务的系统吞吐量。按银行调节机制可集成到现代实时操作系统和硬件设计中，为下一代可预测且高性能的实时计算平台提供基础支撑。
