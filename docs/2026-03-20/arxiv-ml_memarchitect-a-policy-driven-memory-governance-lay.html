MemArchitect: A Policy Driven Memory Governance Layer

arXiv Machine Learning 2026-03-20
Read Original Article →

论文摘要 / Abstract

本文指出持久化大语言模型（LLM）智能体在内存管理方面存在关键治理缺口：标准检索增强生成（RAG）框架将内存视为被动存储，缺乏解决矛盾、执行隐私保护和防止过时信息（"僵尸记忆"）污染上下文窗口的机制。为此，作者提出 MemArchitect——一种将内存生命周期管理与模型权重解耦的治理层，通过显式的基于规则的策略实现内存衰减、冲突解决和隐私控制。实验表明，受治理的内存持续优于无管理的内存，凸显了结构化内存治理对于可靠安全自主系统的必要性。

方法概述 / Method

MemArchitect 采用策略驱动的架构设计，将内存治理逻辑从 LLM 权重中分离出来，形成独立的治理层；该层通过可配置的规则引擎执行内存生命周期管理，包括基于时间的衰减机制、语义冲突检测与消解算法，以及细粒度的隐私访问控制策略。

实验结果 / Results

实验结果表明，在智能体（agentic）应用场景中，经 MemArchitect 治理的内存系统在一致性和可靠性方面持续优于无管理的基线系统；具体而言，治理机制有效减少了僵尸记忆带来的上下文污染，并在多轮交互中保持了更高的事实准确性和隐私合规性。

应用价值 / Applications

MemArchitect 适用于需要长期记忆能力的自主 AI 系统，如个人助理、客户服务智能体和医疗咨询助手等场景；其策略驱动的治理框架为构建可审计、可解释且符合隐私法规（如 GDPR）的持久化智能体提供了关键技术支撑，对提升 AI 系统的安全部署具有重要实践意义。
