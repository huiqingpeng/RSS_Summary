【论文摘要 / Abstract】
本文重新审视了计算机架构研究中广泛使用的SimPoint采样技术，发现标准SimPoint配置（如20个采样点）可能导致显著的估计误差，部分SPEC CPU 2017整数程序的性能预测误差高达40-60%。作者将SimPoint重新表述为分层抽样问题，推导出保守置信区间，并提出了一种两阶段分层采样方法，将最大单应用误差降至3%，同时相比简单随机抽样实现了数量级的样本量减少。

【方法概述 / Method】
论文将SimPoint分解为两个核心组件——分层（聚类）和样本单元选择（质心选择），并分析它们对准确性的独立影响。在此基础上，提出两阶段（双重）采样方案：第一阶段通过大规模初步随机样本改进分层结构，第二阶段进行更具代表性的区域选择，从而实现更精确的统计推断。

【实验结果 / Results】
实证分析表明，传统20个SimPoints配置下，两个SPEC CPU 2017整数应用出现40-60%的性能预测误差；而采用两阶段分层采样后，最大单应用误差降至3%。该方法在保持紧致分析置信区间的同时，相比简单随机抽样将所需样本量减少了一个数量级。

【应用价值 / Applications】
该研究为计算机架构仿真提供了统计基础更严谨、效率更高的实用路径，可广泛应用于处理器设计验证、性能评估和架构探索等场景，帮助研究人员在显著缩短仿真时间的同时获得可靠的性能预测结果。
