【论文摘要 / Abstract】
本文提出了VERINA（可验证代码生成评测平台），首个针对可验证代码生成任务的综合性评测基准，涵盖代码生成、形式化规约生成和证明生成三个核心组件及其组合任务。该基准包含189个手工整理的Lean语言编程任务，配备详细问题描述、参考实现、形式化规约和完整测试集。对当前最先进大语言模型的评测揭示了可验证代码生成领域的重大挑战，尤其是证明生成环节表现薄弱，最佳模型OpenAI o3的证明成功率仅为4.9%。

【方法概述 / Method】
VERINA采用模块化评测框架，将可验证代码生成任务分解为代码正确性、规约的完备性与可靠性、以及形式化证明生成三个独立可评估的维度。基准构建过程中，作者团队手工设计了覆盖多种算法和数据结构的编程任务，并为每个任务配备了经过验证的参考实现、形式化规约（前置条件、后置条件、循环不变式等）以及可自动判定的测试套件。

【实验结果 / Results】
在VERINA基准上对多个SOTA大语言模型的评测显示：OpenAI o3在代码正确性上达到72.6%，规约的完备性与可靠性为52.3%，但证明成功率仅4.9%（单尝试）；其他模型表现更弱，证明生成成为当前可验证代码生成的核心瓶颈。实验结果明确表明，现有LLM在形式化定理证明和验证领域仍有巨大提升空间。

【应用价值 / Applications】
VERINA为高可靠性软件自动生成提供了标准化评测工具，可直接用于指导LLM在关键系统（如航空航天、医疗设备、金融系统）代码生成中的能力改进。该基准填补了形式化验证与代码生成交叉领域的评估空白，有望加速可证明正确代码的自动化生成技术发展，降低关键软件的人工验证成本。
